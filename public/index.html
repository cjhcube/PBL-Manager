Info

What is PBL?

PBL (Permute Both Layers) is a recently developed alg set for Square-1 which combines CP and EP into one step. Both layers are permuted at the same time, using 967 algorithms.

Why use PBL?

PBL's move count is significantly less than CP+EP. The highest possible slice count is 10 (11 with equator) for PBL, and 15 (16 with equator) for CP/EP. This means you are saving about 5 slices each time, equivalent to skipping CP every solve.

How should I learn PBL?

967 algs might seem like a daunting number, but it can be cut down. Lots of PBLs are 2-alg combos or cancellations between 2 shorter algs. Normal algs can be learned using Karnaukh notation, which translates Square-1 algs to be easier to remember.
